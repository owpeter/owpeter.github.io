3.2 版本回退

  • 在Git中，当前版本表示为 HEAD，前一个版本为 HEAD^，上上个版本为 HEAD^^，前n个版本表示为 HEAD~n
  • 接下来，使用以下命令进行版本回退：

其中option有三种：

  • --hard回退到上个版本的已提交状态
  • --soft回退到上个版本的未提交状态
  • --mixed回退到上个版本的已添加但未提交状态

  • 加入回退错了版本，如何回到未回退状态呢？

    • 首先，我们需要知道以前commit的版本号。通过 git log -pretty=oneline查看每次的版本号。
    • 将 head指针移动至为回退时 commit的版本
  • 忘记录版本号了怎么办？ 输入 git reflog查看每一次操作的id

3.3 撤销修改

  • 使用 git checkout -- <file>命令，将文件在工作区进行的修改全部撤回
    这里有两种情况：
    • 一种是 <file>自修改后还没有被放到暂存区，现在，撤销修改就回到和版本库一模一样的状态；
    • 一种是 <file>已经添加到暂存区后，又作了修改，现在，撤销修改就回到添加到暂存区后的状态。
  • 使用 git reset HEAD <file>命令，将在暂存区的文件unstage，放回工作区
  • 参考 3.2 版本回退，撤销本地版本库中的操作
